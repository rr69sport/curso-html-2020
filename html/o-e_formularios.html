Inputs para móviles

Estos inputs, aunque son los mismos que se usan para web, en móviles tienen algunas diferencias que por propósitos de experiencia de usuario, tienen ciertas ayudas a la hora de usarlos. Las validaciones básicas que hace el navegador de cada <input/> es la misma tanto en movil como en web.

<input type="search">
Se utiliza para las barras de búsqueda
<input type="tel">
Se utiliza para introducir números de teléfono
<input type="email">
Se utiliza para introducir un email
<input type="password">
Se utiliza para contraseñas
<input type="url">
Se utiliza para introducir URLs

Como se puede ver, en el navegador del ordenador, a simple vista son todos iguales. Pero en móvil tienen algunas diferencias importantes como ya mencionamos.

Como se marca con el cuadrado rojo, el input de tipo search en móvil muestra una lupa en lugar de la tecla o botón de salto de línea (Tecla Enter en los teclados).

En los inputs de tipo teléfono, el teclado cambia a numérico totalmente para facilitar el marcado del número que necesitemos.

Los inputs de tipo password, agregan una llave como muestra la imagen para acceder a el gestor de contraseñas del sistema para facilitar el acceso a las mismas.

Los input de tipo email muestran un @ como muestra el cuadrado rojo. También pueden sugerir correos electrónicos en la parte marcada con el cuadrado amarillo, pero eso depende de configuraciones de usuario.

Lo inputs de tipo URL muestran sugerencias de urls anteriormente visitadas y además como muestra el cuadrado rojo lo que normalmente es la coma o los emojis, se cambia a una barra inclinada ya que las urls se sirven de este símbolo muchísimas veces.

El curso lo dicta Dorian Desings (Canal de YouTube), y tiene la validación de la W3C. El curso se basa en la documentación brindada por la MDN.
