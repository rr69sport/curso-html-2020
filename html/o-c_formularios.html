<button> vs type="button"

A primera vista notamos que para el input poder mostrar un texto, debemos declararle el atributo value="", eso es porque los inputs no tiene etiqueta de cierre como lo tiene el elemento button <button>Texto</button>

El código mostrado arriba, a efecto visual para el usuario no tiene diferencia ninguna como muestra la siguiente imagen.

Sin embargo sí tienen diferencias internas. Veámos un ejemplo:

Si llenamos el campo de texto que muestra ese formulario y le damos al input de tipo botón, veremos que no sucede nada. Pero si le damos al elemento button, veremos que el formulario se vacía y se recarga la página, esto lo explicamos en la primera parte de esta sección formularios, pero además de eso, es también porque un elemento button es por defecto de tipo submit, para que un input tenga el mismo comportamiento, en vez de declararle en el atributo type="" de tipo button, declararlo de tipo submit.

Un input de tipo botón, nos puede servir para hacer algún tipo de operación por javascript en el formulario o cualquier otra cosa que no sea enviar el formulario, aunque podemos generar ese comportamiento por medio de javascript para eso está submit.

El curso lo dicta Dorian Desings (Canal de YouTube), y tiene la validación de la W3C. El curso se basa en la documentación brindada por la MDN.
